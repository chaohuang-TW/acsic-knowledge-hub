# 公開安全規則

1. 僅納入可公開查閱的官方來源，不讀取私人核心 repository 或本機私人檔案。
2. 禁止個案資料、個人資料、內部人名、私人聯絡資訊、未公開簡報與營運數據。
3. 禁止 Token、API Key、密碼、Cookie、私鑰、本機絕對路徑及環境檔。
4. 動態數字必須同時具備資料日期、來源及最後查證日期；不符合者不納入。
5. 缺漏資料以 `null`、待查證或「官方資料未揭露」呈現，不推測補值。
6. GitHub Pages 維持 `noindex`、`robots.txt`，但不把兩者視為權限控制。
7. 發布前執行單元測試、E2E、建置、敏感資訊與禁止內容掃描。

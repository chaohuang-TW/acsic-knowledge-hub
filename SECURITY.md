# 公開展示版資訊安全規則

## 最高優先原則

1. 禁止讀取、複製、同步或連接私人核心 repository 的內容。
2. 禁止建立私人版與公開版之間的自動同步。
3. 禁止真實保證戶資料、個人資料、內部人名、職稱關係、聯絡資訊或私人電子郵件。
4. 禁止未公開公文、會議內容、簡報、工作底稿、政策判斷、財務或業務數據。
5. 禁止 Token、API Key、密碼、Cookie、私鑰、本機環境資訊與絕對路徑。
6. 所有示範紀錄必須同時標示 `DEMO` 與「示範資料」。
7. 虛構資料不得使用真實機構名稱搭配虛構制度數字。
8. 禁止官方 Logo、政府識別、基金會識別、ACSIC 識別與仿官方視覺。
9. 禁止修改或連結 `chaohuang-tw.github.io` 主官網。
10. `noindex`、robots.txt 或沒有首頁連結都不是安全或存取控制。
11. 發布前必須執行 `pnpm secret:scan` 與人工公開檢查。

## 技術防護

- 完全靜態，沒有後端、登入、資料庫、API Key 或付費服務。
- 不含分析追蹤、廣告、Cookie 或不必要的第三方 JavaScript。
- `.gitignore` 排除環境檔、金鑰、測試結果與建置產物。
- GitHub Actions 使用 GitHub 提供的短期權限，不在程式碼保存 Token。
- Pages 只部署目前 repository 的 `dist` 產物。

## 事件處理

若發現疑似機敏資訊，立即停止發布，不建立公開 Issue，不重印原始值。移除資料後重新掃描，必要時由 repository 擁有者撤銷相關憑證。

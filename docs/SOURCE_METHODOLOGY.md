# 官方來源方法論

## 雙語治理

- 官方英文名稱原文保存，不因介面語言而改寫。
- 中文名稱標示 `official`、`research_translation` 或 `pending`。
- 每筆來源記錄 `originalLanguage`，翻譯不取代原始來源。
- 網站執行不依賴機器翻譯服務或外部翻譯 API。

## 來源優先序

1. 機構官方法規、制度文件及年度報告。
2. 機構官方介紹、業務及治理網頁。
3. 政府主管機關官方資料。
4. ACSIC 官方會員索引，用於會員與 Observer 身分核對。

現行正式會員以 CGTMSE 主辦的 2026 ACSIC Conference 名冊為主；Taiwan SMEG 名冊用於交叉查核及 ACGF Observer 證據。名冊 href 與機構官方網站分開驗證。

搜尋引擎只用於定位官方頁面，搜尋摘要不進入資料集。

## 欄位處理

- 每個唯一來源集中保存於 `src/data/sources.json`，包含 `sourceId`、標題、發布者、標準來源類型、層級、原始與最終網址、原始語言、文件／發布日期、章節、查閱日期、可存取狀態及過時警示。
- 「唯一來源」、「機構—來源關聯」與「欄位證據」為三種不同統計，方法詳見 `SOURCE_REGISTRY_METHODOLOGY.md`。
- 每個已填 Level 2 事實透過 `fieldEvidence` 連回來源，並記錄可讀標題、章節、雙語證據摘要與查核日。
- 官方頁面未明示時使用 `null` 或「官方資料未揭露」。
- 保證成數、上限、費率及營運量等動態數字，只有同時具備資料日期、來源及最後查證日期時才可收錄。
- 不把政策融資、信用保險、聯合組織與直接信用保證承作機構視為相同功能。
- 301／302 保存最終網址；403／502 等暫時錯誤標示 `temporarily_unavailable`，不刪除官方來源或改用非官方鏡像。

## 完整度與可信度

- Level 2 完整度按中央標準的適用欄位及實際欄位證據計算，並另受來源深度、種類、可存取性與過時警示約束。
- 可信度分數結合證據涵蓋率、Tier 1 深度與來源多樣性；來源失效或過時會扣分。可信度不是制度成效評分。

正式引用及決策仍應回到原始來源，確認頁面或文件是否更新。
